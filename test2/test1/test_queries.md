# 📋 Тестовые запросы для проверки ИИ-ассистента (MDC — Оборудование)

> **Фокус:** таблицы MDC (оборудование, станки, состояния, простои, программы, расписания).
> HR-таблицы могут участвовать в JOIN, но не как основная тема запроса.

---

## Уровень 1 — Простые (одна таблица)

| # | Вопрос | Ожидаемая таблица | Ключевые поля |
|---|--------|-------------------|---------------|
| 1.1 | Какие станки есть на заводе? | `mdc_machine` | name, num, oeeplan |
| 1.2 | Какие подразделения (цеха) есть в системе MDC? | `mdc_dept` | name |
| 1.3 | Какие состояния станков есть в справочнике? | `mdc_machine_param` | name, short_name, code, priority |
| 1.4 | Какие группы состояний станков существуют? | `mdc_group_param` | name, short_name, plan_value |
| 1.5 | Покажи историю состояний станков | `mdc_monitoring_value_129` + JOIN | start_time, end_time, machine, state |
| 1.6 | Какие программы выполнялись на станках? | `mdc_operating_program_execution_log` + JOIN | name_up, dt_start, dt_end |
| 1.7 | Покажи расписание работы станков | `mdc_shedule_base` + JOIN | week_day_num, smena_num, time_from |
| 1.8 | Покажи настройки контроля параметров | `mdc_technology_control_setting` | name, group_param_id |
| 1.9 | Покажи журнал нарушений контроля | `mdc_technology_control_log` | param_value, dt_start, dt_end |
| 1.10 | Покажи состояния, привязанные к группам | `mdc_machine_param_in_group` | mdc_machine_param_id, mdc_group_param_id |

## Уровень 2 — Средние (JOIN двух таблиц)

| # | Вопрос | Основные таблицы JOIN |
|---|--------|----------------------|
| 2.1 | Покажи станки с указанием их цеха | mdc_machine JOIN mdc_dept |
| 2.2 | Кто и какие программы выполнял на станках? | mdc_operating_program_execution_log JOIN mdc_machine JOIN hr_worker |
| 2.3 | Покажи детали, которые обрабатывались на станках | mdc_operating_program_assembly_unit JOIN mdc_machine JOIN tp_assembly_unit |
| 2.4 | Какие состояния отслеживаются на каждом станке? | mdc_param_in_machine JOIN mdc_machine JOIN mdc_machine_param |
| 2.5 | Нарушения контроля с привязкой к станкам | mdc_technology_control_log JOIN mdc_param_in_machine JOIN mdc_machine |
| 2.6 | Особые даты в расписании (праздники) | mdc_shedule_meta JOIN mdc_machine |
| 2.7 | Типы контроля параметров | mdc_technology_control_type |

## Уровень 3 — Сложные (агрегации, GROUP BY, подзапросы)

| # | Вопрос | Особенности |
|---|--------|-------------|
| 3.1 | Какое общее время простоев у каждого станка? | SUM(state_time), подзапрос через mdc_machine_param_in_group |
| 3.2 | Сколько программ выполнено на каждом станке? | COUNT, SUM, GROUP BY |
| 3.3 | Аналитика по состояниям из rep_monitoring_by_priority | JOIN 3 таблиц |
| 3.4 | Программы → сборочные единицы на каждом станке | 3-way JOIN |
| 3.5 | Настройки контроля с группами и параметрами | 3-way JOIN |
| 3.6 | Станки у которых OEE > 0 | WHERE oeeplan > 0 |

## Уровень 4 — Очень сложные (множественные JOIN, агрегации)

| # | Вопрос | Особенности |
|---|--------|-------------|
| 4.1 | Полная информация о станке: цех, программы, оператор | LEFT JOIN 4 таблиц |
| 4.2 | Загрузка vs простой по цехам | rep_main_analitic, ROUND, расчёт % |
| 4.3 | Станки с наибольшим простоем за месяц | date('now', '-1 month'), SUM, GROUP BY |
| 4.4 | Состояния с группировкой по группам | COUNT, GROUP BY |
| 4.5 | Расписание с интервалами и цехами | 4-way JOIN, ORDER BY |
| 4.6 | Все нарушения контроля с названиями станков | 4-way JOIN, ORDER BY |

---

## Правила валидации

| Критерий | Правило |
|----------|---------|
| **SQL** | Должен **совпадать** с эталонным (после нормализации пробелов и регистра) |
| **Ответ** | Должен содержать данные или корректное сообщение об отсутствии данных |
| **Итог** | Тест ✅ только если SQL совпал. Ответ — дополнительная проверка |
